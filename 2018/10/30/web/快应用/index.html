快应用

发表于 2018-10-30

一、简介

快应用是基于手机硬件平台的新型应用形态，标准是由主流的手机厂商组成的快应用联盟联合制定，包括小米、华为、金立、魅族、vivo、OPPO、一加、中兴、联想、努比亚，来实现10亿部设备上无需安装、即点即用。

三、主要内容

1. 快应用与前端技术的异同点
1. 类似HTML语法，样式对CSS进行了一些裁剪和扩充，
2. 不支持浮动`float`、绝对定位`absolute`、相对定位`relative`等
3. 快应用的标签与HTML5标签含义不同    
    * 显示文本放置于`text`组件，不能放置于`div`组件
    * 基础组件中`a`和`text`组件支持内部嵌套组件，其它不支持
    * `a`的子组件`span`
    * `text`的子组件`a`和`span`
2. 样式书写
1. 盒模型：默认使用border-box模型，暂不支持content-box模型与box-sizing属性
    * `width`包括`padding`和`border`
    * `height`包括`padding`和`border`

2. 长度单位
    * px：与Web不一样，px是参考项目基准宽度，默认是750
    * %：参考父级
    * 计算公式：设计稿1px / 设计稿基准宽度 = 框架样式1px / 项目配置基准宽度

3. 选择器支持
    * id
    * class
    * tag
    * 并列
    * 后代

4. 样式引入
    * 内部
    * 外部
3. 快应用Flex布局
1. 主轴和交叉轴、容器和项目
2. 快应用容器标签仅为`div`、`list-item`、`tabs`
3. 容器和项目各自支持的属性(截止1020版本)
4. 容器属性
    * `flex-direction`（不支持 `*-reverse`）
    * `flex-wrap`
    * `flex-flow`（不支持）
    * `justify-content`（不支持`space-around`）
    * `aline-items`（不支持`baseline`）
    * `align-content`

5. 项目属性    
    * `order`（不支持）
    * `flex-gorw`
    * `flex-shrink`
    * `flex-basis`（和HTML5特性不一致）
    * `flex`（和HTML5特性不一致）
    * `align-self`

6. 快应用Flex布局与HTML5对比总结
