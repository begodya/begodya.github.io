React Navigation 3x开发指南

发表于 2019-01-29

什么是导航器？

App基本不会只由一个页面组成，管理多个页面的呈现、跳转的组件就是导航器。

React Navigation 3x中6种类型的导航器

  1. createStackNavigator：屏幕上方导航栏，iOS右侧滑入（可配置底部滑入），Android底部淡入；
  2. createBottomTabNavigator：屏幕下方标签栏；
  3. createMaterialBottomTabNavigator：屏幕底部的材料设计主题标签栏；
  4. createMaterialTopTabNavigator：屏幕顶部的材料设计主题标签栏，可通过点击或水平滑动来切换；
  5. createDrawerNavigator：抽屉效果，侧边滑出；
  6. createSwitchNavigator：一次只显示一个页面；

可以通过以上6种导航器来创建App，根据应用场景来使用其中一个或是多个组合

Navigation prop reference

应用中每个页面组件都会自动提供navigation prop，包含以下方法用于触发导航操作：

  • navigate：跳转到另一个页面
  • goBack：关闭当前页面
  • dispatch：向路由发送 action
  • state：当前的状态或路由
  • setParams：对路由参数进行更改
  • getParam：获取具有回退的特定参数
  • addListener：订阅导航生命周期的更新
  • isFocused：屏幕是否有焦点
  • dangerouslyGetParent：返回父级导航器
StactNavigator中附加方法
  • this.props.navigation
    • push：推一个新的路由到堆栈
    • pop：返回到堆栈中的上一个页面
    • popToTop：返回到堆栈中最顶层的页面
    • replace：用新路由替换当前路由
    • reset：清除整个导航状态，并将其替换为多个操作的结果
    • dismiss：关闭当前堆栈
DrawerNavigator中附加方法
  • this.props.navigation
    • openDrawer：打开
    • closeDrawer：关闭
    • toggleDrawer：切换

通用API使用方法

使用navigate进行页面跳转
  • navigation.navigate({ routeName, params, action, key})
  • navigation.navigate( routeName, params, action)
    • routeName：要跳转到的页面的路由名，即导航中配置的路由名
    • params：传递给下一个页面的参数
    • action：如果页面是navigator，则是子路由中运行的子操作
    • key：要导航到的路由的可选标识符。如果已存在，将后退到此路由
使用goBack进行关闭当前页面并返回上一个页面
  • goBack(key)
    • key：表示要返回到的页面标识，而不是routeName
    • 可以通过navigation.state.key来获取页面标识
    • key非必传，也可为null
使用dispatch进行向路由发送action
  • dispatch(action)：给当前界面设置action，会替换原来的跳转，回退等事件
使用state获取当前的state或route
  • 页面可以通过this.props.navigation.state访问其路由
使用setParams对路由的参数进行更改
  • setParams(params)：触发setParams方法允许页面更改路由中的参数，可以更新当前页面的标签和按钮
使用getParam获取指定的参数，可设置获取失败的返回值
使用addListener订阅导航生命周期的更新
  • React Navigation将事件发送到订阅了它们的页面组件：
    • onWillFocus：页面将获取焦点
    • onDidFocus：页面已获取焦点
    • onWillBlur：页面将失去焦点
    • onDidBlur：页面已失去焦点
使用isFocused进行查询页面是否获取到焦点
  • let isFocused = this.props.navigation.isFocused() * 如果页面已获取焦点，返回true，否则返回false
